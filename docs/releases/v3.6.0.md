## Summary

v3.6.0 ships the PR #24 verified plugin apply-back surface plus the Logic 12.2 AX readback hardening from PR #54 / issue #59.

The headline change is a new public MCP tool, `logic_plugins`, with three HC v2 commands:

- `get_inventory` reads physical insert-slot inventory from Logic's AX surface.
- `insert_verified` inserts an allowlisted stock plugin into an exact physical insert slot and returns State A only after inventory readback observes the requested plugin at the requested slot.
- `set_param_verified` writes Compressor `threshold` through an open plugin window and returns State A only after AX readback confirms the requested normalized value.

This is a minor release because the public tool surface grows from 8 write tools to 9 and clients can now use a verified stock-plugin apply-back workflow without relying on legacy Scripter send-only parameter writes.

## Added

- New `logic_plugins` tool with HC v2 envelopes (`state`, `hc_schema: 2`, explicit State C `verified:false`).
- Physical insert-slot inventory with slot read status (`empty`, `ok`, `unreadable`) and `complete` gating.
- Exact-slot plugin insertion through the requested slot's own popup, with popup-anchor verification before any plugin leaf is clicked.
- Live-verified insert allowlist for `Gain`, `Channel EQ`, and `Compressor`.
- Live-verified Compressor `threshold` parameter write/readback using normalized `0..100` AX slider values.
- Canonical verified-plugin identity resolution through `VerifiedPluginCatalog`.

## Changed

- `logic_mixer.insert_plugin` remains available for backward compatibility, but new apply-back workflows should use `logic_plugins.insert_verified`.
- `logic_mixer.set_plugin_param` is documented as legacy unverified State B because Scripter can send values but cannot read them back.
- Documentation and packaging now point at the published `v3.6.0` stable release line; `v3.5.0` remains available as the previous stable tag.

## Fixed

- `logic://tracks` no longer falls into placeholder mode on Logic Pro 12.2 / macOS 26 when Logic exposes the track-header rail through language-neutral AX structure instead of stable localized labels.
- Track type inference now scans header descendants across AX description/title/identifier/help/value fields.
- Transport cycle readback now uses the same Logic 12 control-bar path as verified toggle operations.
- Homebrew helper asset install paths remain guarded against formula/tarball layout drift.

## Migration Notes

- Clients using only the v3.5.0 read resources do not need to change.
- Clients doing plugin apply-back should migrate from `logic_mixer.set_plugin_param` and `logic_mixer.insert_plugin` to `logic_plugins.get_inventory`, `logic_plugins.insert_verified`, and `logic_plugins.set_param_verified`.
- All mutating `logic_plugins` calls require `mode:"duplicate_applyback"` and `project_expected_path`.
- `set_param_verified` currently supports only Compressor `threshold`. Unsupported params fail closed before writing.
