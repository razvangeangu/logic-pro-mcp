## Breaking Changes

The dispatcher input surface is now uniformly fail-closed. Callers upgrading from v3.4.x must review the full enumeration in `CHANGELOG.md` [3.5.0] `### ⚠️ BREAKING`; highlights:

- `send_pitch_bend` value is strictly unsigned `0..16383` (center `8192`); signed aliases removed.
- `import_file` path is sandboxed to regular `.mid` files under `/tmp/LogicProMCP/`.
- `goto_bar`, `set_tempo`, `set_cycle_range`, and `mmc_locate` require explicit parameters (silent defaults removed).
- `play_sequence` rejects top-level `channel` and enforces realtime timing caps on both ports.
- Mixer `set_*`/`insert_plugin` enforce strict ranges and reject conflicting parameter aliases.
