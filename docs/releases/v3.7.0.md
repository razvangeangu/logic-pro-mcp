## Summary

v3.7.0 is the accumulated release for the feature and bug-fix backlog that landed after v3.6.0. It publishes the current 10-tool / 18-resource / 11-template source tree and closes the v43 demo QA set, the issue #60 EN/KO locale-agnostic UI epic, and all open GitHub issues.

The highest-impact changes are setup doctor/lifecycle commands, audio artifact analysis, project audit/export/session workflows, stock instrument and Session Player planning catalogs, locale-agnostic AX policy, stricter track/mixer/transport/project verification, and fail-closed public-demo gates.

## Added

- `LogicProMCP doctor` / `doctor --json` read-only setup diagnostics and remediation anchors.
- Install/update/uninstall dry-run lifecycle planning commands.
- `logic_audio.analyze_file` exported-audio verification.
- `logic://project/audit` and `logic://project/cleanup-plan`.
- Guarded `cleanup_apply` for reversible cleanup-plan steps.
- Export planning plus `export_run` / `export_resume`.
- Stock instrument and Session Player catalogs.
- Planning-only session workflow plans from prompt templates.
- EN/KO `AXLocalePolicy` token bags for UI controls, menus, dialogs, mixer/plugin labels, and region classifiers.
- Demo media guards: bounce guard, asset guard, capture finalize, press-any timeout, play timeout, don't-save modal handling, render fail-closed gates, contact sheet, and evidence manifest.
- GitHub issue templates, PR template, label taxonomy, registry metadata, and MCP publish workflow metadata.

## Fixed Or Hardened

- `logic://project/info.trackCount` now derives from trusted live track cache when saved metadata is name-only or lacks a positive count.
- Track create/select/mutate paths now preserve honest State B/C when UI occlusion prevents verified readback.
- Track mute/solo/arm uses AX checkbox locators with read-back polling.
- `set_instrument` now fails on unmet Library panel, GM Device, External MIDI, and bounded navigation preconditions.
- MIDI import handles `/private/tmp` aliasing, open-dialog polling, `dialog_not_found`, and GM-device audibility downgrades.
- `record_sequence` does not report false success after import failure.
- `project.save` verifies file evidence; `project.save` fails fast for untitled docs; `project.new` unwraps observed names from runScript JSON envelopes.
- `transport.pause` routes through working stop channels before MMC; `set_cycle_range` is documented/demoted when unsupported.
- `goto_position` and `goto_bar` stop carrying contradictory readback notes.
- `set_zoom` verifies via writable horizontal zoom slider and rejects invalid levels.
- Mixer volume/pan writes use AX same-surface readback; master-volume returns honest State B on surface limitations.
- Hidden-mixer plugin inventory readback and mixer reveal paths are more robust.
- `nav.rename_marker` returns typed `not_implemented` State C.
- Bounce flow verifies the settings dialog before confirming and no longer treats missing record-enabled track as a shipped-bounce precondition.
- Demo rendering rejects silent bounces, failed bounce guards, stale issue coverage, invalid raw MP4, non-Logic crops, and non-streamable output.
- Fresh Logic bootstrap handles localized/free-tempo/don't-save dialogs, project picker confirmation, first software-instrument track creation, and no-System-Events UI readiness.
- MCU echo tests are deterministic under parallel load.
- AX `.contains` / `.prefix` matching is diacritic-sensitive where correctness requires it.
